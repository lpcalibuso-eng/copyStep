┌─────────────────────────────────────────────────────────────────────────────┐
│                        📊 API ENDPOINTS (5 ROUTES)                         │
└─────────────────────────────────────────────────────────────────────────────┘

  PUBLIC ROUTES (No Authentication Required)

  1. POST /api/otp/send
     ├─ Request:  { email, firstName, lastName, password, role_id }
     ├─ Response: { success: true, message: "OTP sent successfully" }
     └─ Purpose:  Generate and send OTP to email

  2. POST /api/otp/verify
     ├─ Request:  { email, otp }
     ├─ Response: { success: true, user: {...}, profile_completed: false }
     └─ Purpose:  Verify OTP and create user account

  3. POST /api/otp/resend
     ├─ Request:  { email }
     ├─ Response: { success: true, message: "New OTP sent" }
     └─ Purpose:  Generate and resend OTP code

  ─────────────────────────────────────────────────

  PROTECTED ROUTES (Authentication Required)

  4. POST /api/profile/complete
     ├─ Request:  { phone, profileData }
     ├─ Response: { success: true, user: {...}, profile_completed: true }
     └─ Purpose:  Complete user profile after registration

  5. GET /api/profile/status
     ├─ Request:  (no body)
     ├─ Response: { success: true, profile_completed: true, user: {...} }
     └─ Purpose:  Check if profile completion is done


┌─────────────────────────────────────────────────────────────────────────────┐
│                          🔄 REGISTRATION FLOW                              │
└─────────────────────────────────────────────────────────────────────────────┘

    USER REGISTRATION FORM
           │
           ↓
    Submit with Email & Password
           │
           ├─→ POST /api/otp/send
           │
           ↓
    OTP Generated & Sent to Email
           │
           ├─→ Cached (10-min expiration)
           │
           ↓
    User Receives OTP via Email/Logs
           │
           ├─→ Enters 6-digit OTP
           │
           ↓
    Submit OTP Verification
           │
           ├─→ POST /api/otp/verify
           │
           ↓
    ✅ EMAIL VERIFIED
    ✅ USER ACCOUNT CREATED IN DATABASE
       ├─ avatar_url = Gravatar URL
       ├─ profile_completed = false
       └─ Auto-logged in
           │
           ↓
    PROFILE COMPLETION MODAL SHOWS
    ├─ Display verified email
    ├─ Show Gravatar profile picture
    ├─ Phone number input (required)
    └─ Skip button (for later)
           │
           ├─→ User enters phone number
           │
           ↓
    Submit Profile Completion
           │
           ├─→ POST /api/profile/complete
           │
           ↓
    ✅ PROFILE MARKED COMPLETE
    ✅ Role-specific profile created
       ├─ Student → student_csg_officers table
       └─ Teacher → teacher_adviser table
           │
           ↓
    REDIRECT TO DASHBOARD
    ├─ Student → /dashboard/student
    ├─ Teacher → /dashboard/adviser
    └─ Other → /user
           │
           ↓
    ✅ REGISTRATION COMPLETE - USER LOGGED IN

│                        🛡️  SECURITY FEATURES                              │
└─────────────────────────────────────────────────────────────────────────────┘

  ✅ CSRF Protection
     └─ All requests validated with CSRF tokens

  ✅ Email Verification

  ✅ Password Hashing

     └─ 10-minute cache expiration

  ✅ Cache-Based Storage
     └─ OTP never stored in database

  ✅ Automatic Login
     └─ Only after email verification complete

  ✅ Unique Email Constraint
     └─ Prevents duplicate user accounts

  To enable actual email sending instead of logging:

  1. Update .env:
     MAIL_DRIVER=smtp
     MAIL_HOST=smtp.gmail.com
     MAIL_PORT=587
     MAIL_USERNAME=your-email@gmail.com
     MAIL_PASSWORD=your-app-password
     MAIL_ENCRYPTION=tls
     MAIL_FROM_ADDRESS=noreply@stepplatform.com

  2. Uncomment Mail section in OTPController.php
